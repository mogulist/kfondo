# 네이버맵 → 카카오맵 변경 이유

라이딩 경로 상 보급소·숙박업소를 빠르게 찾는 기능을 구현할 때, **지도/장소 검색 API**로 네이버맵 대신 **카카오맵(카카오 Local API)**을 쓰기로 한 이유를 정리한 문서입니다.

---

## 1. 요구사항

- 사용자가 선택한 **라이딩 경로(폴리라인)**를 기준으로,
- 경로 **주변**의 **보급소(편의점 등)**와 **숙박업소**를 검색해 지도에 표시해야 함.
- 즉, **특정 위도·경도(또는 경로 상의 여러 지점) 주변**의 장소를 찾는 기능이 핵심.

---

## 2. 핵심 차이: “좌표만으로 검색 가능 여부”

### 카카오맵 (카카오 Local API)

- **위도·경도만 있으면** 바로 “이 좌표 주변 Nm 이내 장소” 검색 가능.
- **reverse geocoding(좌표 → 주소/지역명 변환)이 필요 없음.**
- 경로를 일정 간격으로 샘플링한 각 점의 좌표만 넘기면, 해당 점 주변의 편의점·숙소를 바로 조회할 수 있음.

### 네이버맵 (NCP Maps + 네이버 Search API)

- **NCP Maps**: Geocoding, Reverse Geocoding, Directions 등은 제공하지만,  
  “좌표를 중심으로 주변 POI(장소) 검색”하는 REST API 스펙이 공개 문서에서 확인되지 않음.
- **네이버 Search API – 지역 검색**: **텍스트 검색 전용**.  
  예: `query=강릉 편의점`처럼 **검색어**를 넘겨야 하며, **좌표나 반경(radius) 파라미터가 없음**.
- 따라서 “이 좌표 주변”을 쓰려면  
  1) **reverse geocoding**으로 좌표 → 지역명(예: “강릉시 OO동”) 변환 후  
  2) 그 지역명을 검색어에 넣어 Search API 호출  
  이렇게 **2단계 호출**이 필요하고, 호출 수와 비용이 늘어남.

**정리**: “어떤 지점(좌표) 주변” 검색이라는 우리 요구사항에는 **카카오가 구조적으로 맞고**, 네이버는 좌표를 직접 쓰기 어려움.

---

## 3. 기능 비교

| 항목 | 카카오 Local API | 네이버 (NCP Maps + Search 지역) |
|------|------------------|----------------------------------|
| 좌표만으로 주변 검색 | 가능 (`x`, `y`, `radius`) | 불가. 텍스트 쿼리만 지원 → reverse geocoding 선행 필요 |
| 경로 주변 검색 | 가능. 경로 샘플링 후 각 점에서 반경 검색 | 이론상 가능하나, 점마다 reverse geocoding + 검색 2회 호출 |
| 카테고리 필터 | 가능 (편의점 `CS2`, 숙박 `AD5` 등 18종) | Search API에 카테고리 코드 없음. 검색어로만 구분 |
| 거리순 정렬 | 가능 (`sort=distance`, `distance` 필드 제공) | 불가. 정확도순·리뷰수순만 지원 |
| 한 번에 받는 결과 수 | 페이지당 15개, 최대 45페이지 | **최대 5개, 페이징 불가** (display 최대 5, start 최대 1) |
| reverse geocoding | 장소 검색 시 **불필요** | **필수** (좌표 → 지역명 변환) |

- **display 최대 5, start 최대 1**  
  네이버 지역 검색 API는 “한 번의 검색으로 최대 5개 결과만 반환”하고, 2페이지 이후를 가져오는 페이징이 불가능합니다.  
  경로를 따라 여러 보급소/숙소를 넉넉히 보여주려면 부족한 제약입니다.

---

## 4. 비용·운영 관점

- **카카오**: 무료 쿼터(일 10만 건 등)와 초과 시 **호출당 단가(0.5원/2원)**가 문서에 명시되어 있어, 트래픽 증가 시 비용 예측과 예산 통제가 상대적으로 수월함.
- **네이버**: NCP Maps 무료 구간(대표 계정 한정)은 있으나, 초과 구간 요금이 공개 페이지에 `-`로만 표시되는 등, “누구나 쓸 수 있는 공개 서비스”로 트래픽이 늘었을 때 비용이 불명확함.
- reverse geocoding을 **필수**로 쓰는 네이버 방식은, 같은 동작을 구현할 때 **호출 수가 늘어나** 비용이 더 커질 가능성이 있음.

---

## 5. 결론

- **경로 상 지점(좌표) 주변의 보급소·숙소 검색**이라는 요구사항에 맞는 API는 **카카오 Local API**가 명확히 적합함.
- 네이버는 좌표 기반 주변 검색·카테고리·거리 정렬·충분한 결과 수를 한 API로 충족하기 어렵고, reverse geocoding 의존과 호출 제한(5건/페이징 불가) 때문에 같은 서비스를 구현하기에 불리함.
- 따라서 **지도/장소 검색 기능에는 네이버맵 대신 카카오맵(카카오 Local API)을 사용하기로 함.**
